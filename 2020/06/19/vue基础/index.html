    简介

    Vue.js（读音 /vjuː/, 类似于 view） 是一套构建用户界面的渐进式框架。

    渐进式意味着你可以将vue作为应用的一部分嵌入其中，带来更丰富的交互体游。

    Vue 只关注视图层， 采用自底向上增量开发的设计。

    Vue 的目标是通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件。

    vue全家桶：core+vue-router+vuex

    Common Gateway Interface

    CGI（一个协议） 是web服务器和外部应用程序（cgi程序）之间的接口标准，是cgi程序和web服务器之间传递信息的规程。

    MVVM模式

    MVVM最早由微软提出来，它借鉴了桌面应用程序的MVC思想，在前端页面中，把Model用纯JavaScript对象表示，View负责显示，两者做到了最大限度的分离。

    把Model和View关联起来的就是ViewModel。ViewModel负责把Model的数据同步到View显示出来，还负责把View的修改同步回Model。

    MVVM的设计思想：关注Model的变化，让MVVM框架去自动更新DOM的状态，从而把开发者从操作DOM的繁琐步骤中解脱出来！

    性能

    在中大型应用中使用vue可以提高效率，但在小型应用中使用vue反而会降低效率。

    vue的优点：可以帮助我们减少不必要的DOM操作（虚拟DOM），提高渲染效率，进行双向数据绑定

    在小型应用中使用jquery会比较快

    v-show和v-if的区别：

    v-if如果不是true,那么就是整个没有了，而v-show只是把元素隐藏起来。

    v-show不支持template元素，但是v-if支持template元素。

    v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。

    相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。

    一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

    使用这个指令可以用于解决插值表达式的“闪动”问题（一般是因为浏览器太卡了）

    用法：提供样式；在插值表达式中所在的标签中添加v-clock指令

    [v-clock]{

    }

    原理：通过样式隐藏内容，之后在内存中进行值替换，替换好之后再显示最终的值

    填充纯文本，不会有闪动问题

    计算属性和方法的区别
    1. 计算属性是基于依赖进行缓存的
    2. 方法不存在缓存
    3. 方法在每次调用时都会执行，而计算属性，如果里面的值没有改变，那么除了第一次会执行，之后就不会执行了，除非依赖的数据改变了
