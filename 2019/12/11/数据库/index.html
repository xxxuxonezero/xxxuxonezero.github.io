    数据库概念

    DB：database 数据库。保存了一系列有组织的数据

    DBMS：数据库管理系统。数据库通过DBMS创建和操作的容器（MySQL、Oracle、DB2、sqlserver)

    SQL:结构化查询语言，用于和DBMS通信，补充某个数据库软件特有的

    MySQL服务的启动与停止

    MySQL服务的登录

    mysql [-h 主机名 -P 端口号] -u 用户名 -p（密码）

    MySQL常见命令

    查看当前所有数据库：mysql> show databases;
    打开指定库：mysql> use 库名;
    查看当前库所有表：mysql> show tables;

    查看其他库所有表：mysql> show tables from 库名

    创建表：mysql> create table 表名(
    -> id int,
    -> name varchar(20));

    查看表结构：mysql> desc 表名;

    查看版本：select version();

    DQL语言（数据查询）

    基础查询

    select 查询列表 from 表名

    1. 查询列表可以使表中的字段、常量值、表达式、函数

    2. 查询结果是一个虚拟的表格

    起别名：

    1. as
    2. 空格

    去重：distinct

    +的作用：（只是运算符）

    1. 数字+数字
    2. 字符+数字：尝试将字符转为数值进行加法，若不行，则是字符为0进行加法
    3. null+数字：结果为null

    函数：

    • concat():拼接字符串
    • ifnull():判断字段或表达式是否为null,若是，则返回指定的值，否则返回原本的值

    ####条件查询

    select 查询列表 from 表名 where 筛选条件；

    • 按条件表达式（>,<=,<>,>=,<=)

    • 按逻辑表达式（and ,or,not)

    • 模糊查询（like,between and,in,is null，is not null)

      通配符：

      %：表示任意个数的字符

      _:任意单个字符

      ==between and 包含临界值==

      ===不能用于判断null==

      is null 和 <=>的比较：
      is null 可读性高，is null 只可以判断null,<=>可以和普通类型的数值比较是否等于

      like 可以模糊匹配字符型数据，也可以是数值型

    排序查询

    语法：select 【】 from 【】 （where 【】） order by 【】

    特点：

    1. asc：升序；desc:降序，默认是升序
    2. order by子句中可以支持单个字段、多个字段、表达式、函数、别名
    3. order by一般放在查询语句的最后面，limit子句除外

    常见函数

    概念：类似于java中的方法

    调用：select 函数名（实参列表） 【from 表】

    分类：

    • 单行函数（concat(),llength())
    • 分组函数（做统计使用）:又称聚合函数，统计函数、组函数
    单行函数

    1、字符函数：

    • length：获取字符的字节数（一个汉字3个字节）

    • concat：拼接字符串

    • upper,lower

    • substr,substring：截取从指定索引处指定长度的字符，索引从1开始

    如：select concat(upper(substr(last_name,1,1)),’_’,lower(substr(last_name,2))) output from employees;

    • instr:返回子串第一次出现的起始索引，若不存在则返回0

    • trim:去除前后的指定字符，若不指定则默认为空格

      ​ select length(trim(‘ 张翠山 ‘)) as output;
      ​ select trim(‘a’ from ‘aaaaaaaaaaa张aaaaaaaaaa翠山aaaaaaaaa’)

    • lpad:用指定的字符实现左填充指定长度

      ​ select lpad(‘殷素素’,9,’*’)

    • rpad

    • replace:替换

      ​ replace(string,被替换的字符，用于替换的字符)：全部替换

    2、数学函数：

    • round:四舍五入，默认得到的是整数 round(x)/round(x,小数点后保留的位数)
    • cell:向上取整，返回>=该参数的最小整数
    • floor:向下取整，返回<=该参数的最大整数
    • truncate:截断 truncate(x,小数点后的位数)
    • mod：取余
    • rand:获取随机数，0~1之间

    3、日期函数

    • now():返回当前系统时间（日期+时间）

    • curdate():返回当前系统日期，不包含时间

    • curtime():当前时间，不包含日期

      ​ 获取指定部分：年、月、日、小时、分钟、秒

      ​ year(时间),month(时间),day(时间)……

    • str_to_date:字符->日期

      ​ select str_to_date(“09-13-1999”,”%m-%d-%Y”) //1999-09-13

    • date_format:日期->字符 select date_format(now(),’%Y年%m月%d日’)

      ​ select last_name,date_format(hiredate,’%m月/%d日 %y年’) from employees

      ​ where commission_pct is not null;

    4、其他函数

    • select version()
    • select datebase()
    • select user()
    • select md5(‘字符‘）：返回md5加密形式

    5、流控制函数

    • if函数：相当于if…else…

      ​ select if(10<5,’小’,’大’) //大

    • case函数：

    ​ 1、switch case（等值判断）

    ​ 2、类似于多重if（区间判断）

    实践：

    分组函数
    • sum:求和
    • avg：平均值
    • max：最大值
    • min：最小值
    • count：计算个数

    sum,avg处理数值型；min,max,count可以处理任何类型的值

    分组函数都忽略null值

    可以和distinct搭配

    select count(distinct salary) from employees

    和分组函数一同查询的字段有限制，要求是group by后的字段，因为分组函数返回的是一个值

    count详细解释：
    count(*),count(1):个数为表中记录的个数

    ==一般使用count(*)计数==

    计算最大入职日期和最小入职日期相差的天数

    select datediff(max(hiredate),min(hiredate)) from employees

    分组查询

    语法：select 分组函数，列（要求出现在group by后面） from 表 【where 筛选条件】group by 分组的列表【order by 子句】

    1、按表中单个字段分组

    2、按表中多字段分组（必须满足这些字段完全一致才算是一组）

    如何添加分组后的筛选条件？(利用having关键字，放在group by 后面)

    特点：

    • 分组查询中筛选条件分为两类：分组前筛选和分组后筛选

      分组函数做条件一定是放在having子句中

      能用分组前筛选的就优先使用分组前筛选，对性能有好处

    • 支持单字段分组，多字段分组（多字段间用逗号隔开，无顺序要求），表达式或函数（用的较少）

    • 也可以添加排序（放在分组查询的最末尾）

    连接查询（多表查询/多表连接）

    当查询字段来自多个表时，使用连接查询。

    笛卡尔乘积现象：表1 有m行，表2 有n行，结果m*n行

    发生原因：没有有效的连接条件

    如何避免：添加有效的连接条件

    select name,boyname from beauty,boys where beauty.boyfriend_id=boys.id;

    按年代分类：

    • sql92标准：仅支持内连接
    • sql99标准：支持内连接+外连接（左外+右外）+交叉连接

    按功能分类：

    • 内连接(sql92语法）

      • 等值连接

        1. 多表等值连接的结果为多表的交集部分

        2. n表的连接，至少要有n-1个连接条件

        3. 多表的顺序没有要求

        4. 一般要为表起别名

        ·select last_name,department_name,commission_pct from employees e,departments d where commission_pct is not null and e.department_id=d.department_id

      • 非等值连接

      • 自连接

    • 外连接（分主、从表）

      用于查询一个表中有另一个表中没有的记录

      外连接查询结果为主表中的所有记录，如果从表中有和它匹配的，则显示匹配的值；否则显示null

    • 左外连接

      左边是主表

    • 右外连接

      右边是主表

    • 全外连接（full outer join)

      ==mysql中无法使用全外连接==

      全外连接=内连接的结果+表1中有但表2没有的+表2中有但表1没有的

    • 交叉连接（cross join):笛卡尔乘积

    ==如果为表起了别名，那么查询的字段就不可以用原来的表名去限定了。==

    子查询（内查询）

    出现在其他语句中的select语句，称为子查询或内查询
    外部的查询语句，称为主查询或外查询

    分类：

    • 按子查询位置：

      • select后

        仅支持标量子查询

      • from后

        支持表子查询

      • where或having后

        支持标量子查询（单行）、列子查询（多行）、行子查询

        特点：

        1. 子查询放在小括号内
        2. 子查询一般放在条件的右侧
        3. 标量子查询一般搭配着单行操作符使用（> < >= <= = <>)
        4. 列子查询一般搭配着多行操作符使用（in/not in,any/some,all)
        5. 子查询先于主查询执行

    • exists后（相关子查询）

      支持表子查询

      exists 用于判断子查询中是否有值，结果为1或0

    • 按结果集行列数不同

      • ==标量子查询（结果集只有一行一列）==
      • ==列子查询（结果集一列多行）==
      • 行子查询（结果集一行多列）
      • 表子查询（结果集一般多行多列）

    分页查询

    应用场景：要显示的数据一页显示不全， 需要分页提交sql请求。

    特点：

    1. 放在查询语句的最后
    2. 公式：limit (page-1)*size,size

    联合查询（union)

    用处：将多条查询语句的结果合并成一个结果

    应用场景：要查询的结果来自于多个表，且多个表没有直接的连接关系，但查询的信息一致

    特点：

    1. 多条查询语句的列数一致
    2. 要求多条查询语句的列的类型和顺序最好一致
    3. 会自动去重（union all 不会去重）

    DML语言（增删改）

    插入语句

    1. 插入值的类型要与列的类型一致或兼容
    2. 不可以为null的列必须插入值，可以为null的可以省略其列名和对应的值
    3. 列的顺序可以调换
    4. 列数和值的个数必须一致
    5. 可以省略列名，默认是所有列，而且列的顺序和表的列顺序是一致的

    法一与法二的比较：

    1. 法一支持插入多行，而法二不支持

    1. 法一支持子查询，而法二不支持

    删除语句

    单表删除

    一、delete

    二、truncate（表并未被删除，其中的数据全部清除）

    多表删除

    delete和truncate的比较：

    1. delete 可以加where 条件，truncate不能加
    2. truncate删除，效率高一丢丢
    3. 假如要删除的表中有自增长列，
      如果用delete删除后，再插入数据，自增长列的值从断点开始，
      而truncate删除后，再插入数据，自增长列的值从1开始。
    4. truncate删除没有返回值，delete删除有返回值
    5. truncate删除不能回滚，delete删除可以回滚.

    数据类型

    数值型：整型、小数（定点、浮点）

    字符型：较短的文本char、varchar；较长的：text、blob(较长的二进制数据)

    日期型：

    整型

    分类：tinyint、smallint、mediumint、int\integer、bigint

    特点：

    1. 如果不设置有符号还是无符号，那么默认是有符号。若想设置无符号，那么要添加unsigned
    2. 若插入的数值超出了整型的范围，会报错out of range
    3. 若不设置长度，会有默认长度（当zerofill时有效，会使int成为unsigned,若长度不够，会在左侧填充0）
    小数

    分类：
    1.浮点型
    float(M,D)
    double(M,D)
    2.定点型
    dec(M，D)
    decimal(M,D)

    M：整数部位+小数部位
    D：小数部位

    若小数点位数超过D，则四舍五入；若整数位数超出，则报错。

    时间

    • date

      MySQL 以 ‘YYYY-MM-DD’ 格式检索与显示date值。支持的范围是 ‘1000-01-01’ 到’9999-12-31’。

    • datetime

      MySQL 以 ‘YYYY-MM-DD HH:mm:ss’格式检索与显示 DATETIME 类型。支持的范围是’1000-01-0100:00:00’ 到 ‘9999-12-3123:59:59’。(“支持”的含义是，尽管更早的值可能工作，但不能保证他们均可以。)

    常见约束

    约束：一种限制，用于限制表中的数据，为了保证数据的准确和可靠性

    分类：

    • not null:非空
    • default：默认，用于保证该字段有默认值
    • primary key：主键，保证值有唯一性且不为空，一个表中至多有一个主键，但是可以把多个列组合成为一个主键（组合主键）
    • unique：唯一性但可以为空，至多可以有一个为空
    • check：检查约束（mysql中不支持）
    • foreign key：外键，用于限制两个表的关系，用于保证该字段的值必须来自于主表的关联列的值，在从表添加外键约束，用于引用主表中某列的值

    约束的添加分类：

    • 列级约束：
      六大约束语法上都支持，但外键约束没有效果

    • 表级约束：

      除了非空、默认，其他的都支持

      • 语法：【constraint 索引名】 约束（列名）

    外键：

    1. 要求在从表设置外键关系
    2. 从表的外键列的类型和主表的关联列的类型要求一致或兼容，名称无要求
    3. 主表的被引用列必须是一个key（一般是主键或唯一）
    4. 插入数据时，先插入主表，再插入从表；删除时，先从表再主表

    删除约束

    1.删除非空约束
    ALTER TABLE stuinfo MODIFY COLUMN stuname VARCHAR(20) NULL;
    2.删除默认约束
    ALTER TABLE stuinfo MODIFY COLUMN age INT ;
    3.删除主键
    ALTER TABLE stuinfo DROP PRIMARY KEY;
    4.删除唯一
    ALTER TABLE stuinfo DROP INDEX up;
    5.删除外键
    ALTER TABLE stuinfo DROP FOREIGN KEY 索引名

    标识列（自增长列）

    特点：

    • 标识列必须与key搭配
    • ==一个表至多一个标识列==
    • 标识列的类型必须是数值型
    • 可以通过set auto_increment_increment=3,修改步长为3

    定义标识列的方法

    1. 创建表时，设置标识列（auto_increment）

    2. 修改表时设置标识列

      alter table test modify column id int auto_increment;

    3. 修改表时删除标识列

      alter table test modify column id int;

    事务（TCL：事务控制语言)

    事务：一个或一组sql语言组成一个执行单元，这个执行单元要么全部执行，要么全部不执行。

    回滚：撤销之前的操作

    事务的ACID属性：

    • 原子性（aotmicity):事务是一个不可分割的工作单位，要么都执行要么都不执行
    • 一致性（consistency）：使数据库从一个一致性状态变换到另一个一致性状态。
    • 隔离性（isolation）：一个事务的不行不受其他事务干扰
    • 持久性（durability）：一个事务一旦被提交，那么它对数据库中的数据的改变就是永久性的。

    事务的创建

    • 隐式事务：没有明显的开始和结束的标记；insert,update,delete

    • 显示事务：有明显的开始和结束的标记

      ​ 必须先设置自动提交功能为禁用（set autocommit=0)

    并发事务

    1. 并发问题如何发生的？

      多个事务同时操作同一个数据库的相同数据时

    2. 常见的事务并发问题

    • 脏读： 一个事务读取了其他事务还没有提交的数据，读到的是其他事务“更新”的数据
    • 不可重复读：一个事务多次读取，结果不一样
    • 幻读: 一个事务读取了其他事务还没有提交的数据，只是读到的是其他事务“插入”的数据

    1. 如何解决并发问题？

      通过设置隔离级别解决并发问题

    隔离级别

    • read uncommitted：出现脏读、不可重复读和幻读
    • read committed:若事务未提交，那么读到的数据依旧是未更改前的，避免了脏读
    • repeatable read:：避免了脏读和不可重复读
    • serializable:避免了脏读、不可重复读和幻读（使用较少，因为效率低下）

    mysql中默认第三个隔离级别repeatable read

    设置隔离级别

    设置当前连接的隔离级别：set session transaction isolation level …

    设置全局的隔离级别：set global session transaction isolation level …

    查看隔离级别：select @@transaction_isolation

    视图（虚拟表）

    视图：是通过表动态生成的数据，只保存了sql逻辑，不保存查询结果（sql5.1的新特性）

    好处：

    1. 重用sql语句
    2. 简化了复杂的sql操作，不必知道它的查询细节
    3. 保护数据，提高安全性

    ==在视图中插入或修改值会对原始表有影响，不太安全==

    具备以下特点的视图不允许更新：

    • 包含以下关键字的sql语句：分组函数、distinct、group by、having、union或者union all
    • 常量视图
    • where后的子查询用到了from中的表
    • join
    • from一个不能更新的视图

    表和视图的对比

    创建语法的关键字 是否实际占用物理空间 使用
    视图 create view 没有，只是保存了sql逻辑 增删改查，一般不能使用增删改
    表 create table 占用 增删改查

    变量

    • 系统变量：变量由系统提供，属于服务器层面

    • 全局变量

      对所有连接有效

    • 会话变量

      仅仅对当前连接有效

    • 自定义变量

      步骤：声明、赋值、使用

      • 用户变量

        作用域：同于会话变量的作用域

        应用在任何地方

    • 局部变量

      作用域：仅仅在定义它的begin end块中有效
      应用在 begin end中的第一句话

      对比用户变量和局部变量：

    作用域 定义和使用的位置 语法
    用户变量 当前会话 会话的任何地方 必须加@，不用限定类型
    局部变量 begin end中 只能在begin end中，且为第一句话 一般不用加@，需要限定类型

    存储过程和函数

    存储过程：一组预先编译好的sql语句的集合，理解成批处理语句（提高了效率，减少了编译次数和数据库服务器的连接次数）

    注意：

    1. 参数列表包含三部分：参数模式（in,out,inout）、参数名、参数类型

      1. in：该参数可以作为输入，也就是该参数需要调入方传入值
      2. out:该参数可以作为输出，也就是该参数可以作为返回值
      3. inout：该参数可以作为输入也可以作为输入，也就是该参数既需要调入方传入值，又可以返回值

    2. 如果存储体只有一句话，begin end 可以省略

      存储过程体的每条sql语句结尾必须加分号，存储体过程的结尾可以用delimiter重新设置

      实例：

    #####in模式

    create procedure myp2(in beautyname varchar（20）)
    begin
    …
    end
